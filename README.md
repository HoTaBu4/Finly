# React Native План додатку для трекінгу витрат (iOS + Android)

## 1. Ціль продукту
Додаток призначений лише для обліку витрат.

Ключові задачі:
- швидко додавати витрати;
- бачити статистику витрат за період;
- групувати витрати за категоріями;
- спрощувати ввід через голос, чек і AI.

## 2. MVP-функції (обов'язково)
- Додати, редагувати, видалити витрату.
- Статистика витрат: за місяць, за рік, за довільний період (від дати до дати).
- Категорії/групи: створити, перейменувати, змінити порядок 
- Голосовий ввід: користувач озвучує витрату, AI створює чернетку, користувач підтверджує або виправляє.
- OCR по фото чека для автозаповнення. (????????)

## 3. Технології і стек
- React Native + TypeScript (Expo для MVP).
- Навігація: React Navigation.
- Стан: Zustand + TanStack Query.
- Локальна БД: SQLite (`expo-sqlite`) + Drizzle ORM.
- Бекенд: Supabase (Auth + Postgres + Edge Functions) або Firebase.
- Безпечне зберігання: `expo-secure-store`.
- Форми і валідація: React Hook Form + Zod.
- Графіки: `victory-native` або `react-native-gifted-charts`.

## 4. Apple Pay / Google Pay
На практиці додаток не зможе просто прочитати повну історію транзакцій з Apple Pay або Google Pay як банківську виписку.

Реалістичний підхід у MVP:
- зберігати `paymentMethod` (`card`, `cash`, `apple_pay`, `google_pay`);
- суму і магазин брати з голосу, OCR або ручного вводу;
- після AI-автозаповнення завжди просити підтвердження користувача.

## 5. Модель даних
Сутності:
- Expense
- CategoryGroup
- Merchant
- Budget

Поля `Expense`:
- `amount`
- `currency`
- `merchantName`
- `categoryGroupId`
- `paymentMethod`
- `expenseDate`
- `note`
- `source` (`manual` | `voice_ai` | `receipt_ocr` | `import`)
- `aiConfidence`
- `status` (`draft` | `confirmed`)

## 6. Флоу додавання витрати
- Ручний: користувач вводить поля сам.
- Голосовий: speech-to-text -> AI parsing -> чернетка -> підтвердження.
- Чек: фото -> OCR -> AI parsing -> чернетка -> підтвердження.
- Швидкий повтор: останні магазини/суми як шаблони.

## 7. AI-сервіс (через бекенд)
Правило: ключі AI тільки на сервері.

Маршрутизація моделей:
- `OpenAI` для простих запитів (короткі Q&A, базові пояснення, швидкі підказки).
- `Anthropic` для складних запитів (глибший аналіз витрат, довші reasoning-відповіді, складні фінансові сценарії).
- fallback: якщо основний провайдер недоступний, використовувати інший провайдер з позначкою в логах.

Вихід AI у strict JSON:
- `amount`
- `merchantName`
- `paymentMethod`
- `expenseDate`
- `categorySuggestion`
- `confidence`
- `missingFields`

Якщо даних не вистачає, додаток задає короткі уточнення:
- "Яка сума?"
- "Який магазин?"

- "Це Apple Pay, Google Pay, картка чи готівка?"(phrase 2)

## 8. Аналітика витрат
Екрани:
- Dashboard (загальна сума витрат).
- Періоди: місяць, рік, custom range.
- Розбивка за категоріями.
- Тренд порівняно з попереднім періодом.
- Топ магазинів.

Фільтри:
- категорія;
- метод оплати;
- магазин;
- діапазон дат.

## 9. Керування категоріями
- CRUD для категорій.
- Drag-and-drop порядок.
- Merchant rules (наприклад, "Starbucks" -> "Кафе").
- Fallback на AI-класифікацію.
- Manual override: виправлення користувача зберігається як правило.

## 10. Безпека і приватність
- Авторизація (email/OAuth).
- Шифрування локальних секретів.
- HTTPS only.
- Consent-екран для AI та OCR.
- Логи змін AI-пропозицій.
- Опційний App Lock (біометрія/PIN).

## 11. Тестування і реліз
Тести:
- Unit: моделі, категоризація, підрахунки статистики витрат.
- Component: форми вводу, голосовий флоу, підтвердження чернетки.
- E2E: "голос/чек -> AI -> підтвердження -> статистика".

Реліз:
- iOS: TestFlight.
- Android: Google Play Internal Testing.
- CI/CD: GitHub Actions + EAS Build.

## 12. Post-MVP
- Open Banking інтеграції для імпорту транзакцій.
- Повторювані витрати та підписки.
- Бюджети і алерти перевищення лімітів.
- Сімейні/спільні гаманці.
- Мультивалюта та мультимовний OCR.
- Міні-чат для обговорення бюджету і фінансових питань (AI assistant).

## 13. Монетизація (MVP -> Scale)
Основна модель: `Freemium + підписка + AI credits`.

Тарифи:
- `Free`:
  - необмежений ручний облік витрат;
  - базова статистика (місяць/рік);
  - 1 пристрій, без advanced-аналітики.
- `Pro` (рекомендований головний план):
  - 3.99 USD/місяць або 39.99 USD/рік;
  - розширена аналітика (custom range, тренди, топи, фільтри);
  - мультипристрійність + backup/sync;
  - бюджети, алерти, експорт CSV.
- `Family` (Phase 2):
  - 8.99 USD/місяць або 69.99 USD/рік;
  - все з `Pro`;
  - до 5 учасників у спільному гаманці;

Додатково:
- `AI Pack` для Free-користувачів: одноразова покупка додаткових 100 AI-дій.

## 14. Що саме продаємо
Не продаємо "табличку витрат" (це легко замінити), продаємо:
- економію часу (голос і OCR -> готова чернетка);
- зменшення рутини (правила магазинів, автокатегоризація);
- інсайти для рішень (тренди, перевитрати, бюджети);
- спільний фінконтроль (`Family`, Phase 2).

## 15. Paywall-стратегія
Paywall показуємо не на старті, а в момент цінності:
- після вичерпання безкоштовних AI-дій;
- при відкритті advanced-аналітики;
- при спробі увімкнути sync/backup або додати 2-й пристрій;
- при активації сімейного доступу (`Family`, Phase 2).

Рекомендація:
- 7-денний trial для `Pro` (без карти в onboarding не вимагати);
- річний план за замовчуванням (краща окупність і lower churn).

## 16. Технічне впровадження монетизації
Платежі:
- iOS: App Store In-App Purchases.
- Android: Google Play Billing.
- Для MVP найшвидше через `RevenueCat` (єдиний SDK для iOS/Android + webhooks).

Бекенд-сутності:
- `Subscription`:
  - `userId`
  - `tier` (`free` | `pro` | `family`) (`family` = Phase 2)
  - `status` (`active` | `trialing` | `grace_period` | `expired`)
  - `currentPeriodEnd`
- `UsageQuota`:
  - `userId`
  - `periodStart`
  - `periodEnd`
  - `aiUsed`
  - `aiLimit`

Правила доступу:
- feature flags на бекенді (не лише в клієнті);
- перевірка entitlement перед AI-викликом;
- при перевищенні ліміту: м'який блок + екран upgrade;
- webhook-оновлення підписок (renewal, cancel, billing issue).

## 17. Unit Economics і KPI
Цілі монетизації на перші 3 місяці після релізу:
- Free -> Trial conversion: > 8%
- Trial -> Paid conversion: > 35%
- Paid churn (monthly): < 8%
- ARPPU: > 4.5 USD

Фінмодель (чернетка):
- тримати gross margin > 75%;
- якщо середня собівартість 1 AI-дії росте, зменшуємо ліміти або піднімаємо ціну `AI Pack` - 2.99;
- найцінніша аудиторія: користувачі, які додають витрати 4+ дні/тиждень (для них окремі upsell-кампанії).

## 18. Міні-чат для бюджету і фінпитань
Ціль:
- дати користувачу швидкий "фінансовий copilot" прямо в додатку;
- відповідати на питання типу "де я перевищив бюджет?", "як скоротити витрати?", "скільки можу витратити до кінця місяця?".

MVP scope:
- окремий екран `Budget Chat`;
- контекст відповідей: витрати користувача, категорії, бюджети, період;
- пресети запитів (quick prompts) + вільний текст;
- відповіді у форматі:
  - короткий висновок;
  - 2-3 практичні рекомендації;
  - CTA: "перейти в категорію/період".

Guardrails:
- дисклеймер: це інформаційна порада, не інвестиційна/податкова консультація;
- PII redaction у логах;
- ліміт довжини діалогу і rate limit на юзера;
- fallback-повідомлення, якщо бракує даних за період.

Монетизація чату:
- `Free`: до 10 чат-повідомлень/місяць;
- `Pro`: до 300 чат-повідомлень/місяць;
- списання з єдиного AI-ліміту (голос + OCR + чат), щоб тримати просту економіку.
- роутинг запитів: прості питання -> `OpenAI`, складні питання -> `Anthropic`.
